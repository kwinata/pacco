Remote Manager¶

class pacco.manager.remote_manager.RemoteManager¶

Function to manage .pacco_config file as the storage for remote lists With RemoteManager, you can manage multiple Remote s

add_remote(name, configuration)¶

Add/register a new remote. Currently there is two possible configuration:

Local client:

{
    'remote_type': 'local',
    'path': '[PATH]', (optional, will use ~/.pacco/ if not declared)
}

Nexus site client:

{
    'remote_type': 'nexus_site',
    'url': '[URL]',
    'username': '[USERNAME]',
    'password': '[PASSWORD]',
}
Parameters

  • name (str) – the name of the new remote

  • configuration (Dict[str, str]) – a dictionary of the configuration as described above.

Return type

None

default_download(package_name, assignment, dir_path, fresh_download=False)¶

Try to download a package binary from the remotes in the default remote list.

Parameters

  • package_name (str) – package registry name of the binary

  • assignment (Dict[str, str]) – the dictionary of the binary configuration

  • dir_path (str) – the download destination

  • fresh_download (Optional[bool]) – will not use cache if True

Return type

None
